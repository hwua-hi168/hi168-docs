# 空间与存储管理

## 概述

Hi168 平台提供 **命名空间管理** 和 **存储桶管理** 两大核心资源管理能力。命名空间用于隔离不同实验或项目的运行环境与资源，避免相互冲突；存储桶则提供基于 S3 协议的对象存储服务，支持数据的持久化存储、分层计费、分享与跨应用挂载。

存储管理模块包含以下核心功能页面：

| 页面 | 说明 |
| --- | --- |
| **存储桶列表** | 查看和管理所有存储桶，创建、重命名、删除存储桶 |
| **存储桶详情** | 浏览和操作桶内文件，上传、下载、复制、移动、预览、分享文件 |
| **购买存储** | 购买标准存储空间配额，扩展存储容量 |
| **我的分享** | 管理已创建的所有分享链接，编辑有效期、密码、访问限制 |
| **存储任务** | 查看文件复制、移动等异步任务的执行进度 |
| **密钥管理** | 管理 S3 访问密钥，用于第三方工具连接 |

---

## 一、空间管理

### 1.1 功能说明

命名空间（Namespace）是平台中资源隔离的基本单元。每个命名空间可包含特定实验或项目所需的应用、数据及其他资源，确保不同实验之间相互独立、互不干扰。

平台空间分为两类：

| 空间类型 | 说明 |
| --- | --- |
| **个人空间（私有空间）** | 用户专属的私有命名空间，拥有独立控制权，可查看已创建的应用数量及可用硬件资源（CPU / 内存） |
| **公共命名空间** | 可供多人访问和使用的共享空间，适用于团队协作场景 |

### 1.2 个人空间

进入「空间管理」→「我的个人空间」，可查看当前用户在该空间内正在运行的所有应用实例。

**支持的操作：**

| 操作 | 说明 |
| --- | --- |
| **查看应用状态** | 查看应用运行状态与资源详情 |
| **修改应用名称** | 直接编辑应用的显示名称 |
| **调整资源配额** | 修改应用的 CPU / 内存分配 |
| **打开应用** | 直接进入运行中的应用操作界面 |

### 1.3 创建个人空间

用户可在空间管理页面中创建新的个人命名空间，用于隔离不同项目或实验的资源环境。

**创建流程：**

1. 进入「空间管理」页面。
2. 点击「创建空间」。
3. 填写空间名称及相关配置。
4. 确认创建。

## 二、存储管理

### 2.1 功能说明

存储桶（Bucket）是平台提供的基于 S3 协议的对象存储容器，类似于个人网盘，用于存储和组织用户的数据文件。存储管理涵盖存储桶的创建、配置、内容操作、维护和监控等完整生命周期管理，确保数据安全、高效、有序地存储和访问。

### 2.2 存储桶列表

进入「存储管理」后，系统展示当前用户的所有存储桶列表，包括以下信息：

| 字段 | 说明 |
| --- | --- |
| **存储桶名称** | 用户自定义的桶名称 |
| **存储层级** | 标准存储 / 低频存储 / 冷存储 / 归档存储 |
| **最后操作时间** | 最近一次对桶内内容进行操作的时间 |

在存储桶列表页面，还可查看以下概览信息：

- **基本信息**：已用容量、总容量、各层级桶数
- **存储空间组成**：标准层免费配额使用情况，以及含余额折算的可用上限
- **配额与限速**：当前配额信息、用户级限速设置

此外，列表页提供搜索框，可通过关键字快速检索存储桶。

### 2.3 创建存储桶

#### 2.3.1 创建方式

平台提供多种创建存储桶的方式，适配不同使用习惯：

| 方式 | 操作路径 |
| --- | --- |
| **菜单创建** | 在存储桶列表页面点击「新建存储桶」按钮 |
| **右击创建** | 在存储管理界面中右击空白区域选择创建 |
| **桌面创建** | 在云桌面上直接右击创建存储桶 |

#### 2.3.2 创建流程

1. 点击「新建存储桶」。
2. 输入存储桶名称（3–10 个字符，支持中文、英文和数字，不能包含符号或空格）。
3. 选择存储层级（详见下方「存储层级说明」）。
4. 确认创建。

创建完成后，可为存储桶在桌面上创建快捷方式，方便快速访问。

#### 2.3.3 存储层级说明

平台支持四种存储层级，不同层级适用于不同的数据访问频率和成本需求：

| 层级 | 特点 | 适用场景 | 计费方式 |
| --- | --- | --- | --- |
| **标准存储** | 高速读写，3 副本高可靠 | 日常频繁读写的文件 | 配额内免费，超出按量计费 |
| **低频存储** | EC 纠删码存储，更经济 | 月均访问 1 次的数据 | 始终按量计费，单价更低 |
| **冷存储** | 最低存储 60 天 | 季度访问的数据，适合备份归档 | 始终按量计费，单价更低 |
| **归档存储** | 最低存储 180 天，价格最低 | 长期归档保留数据 | 始终按量计费，单价最低 |

> **⚠️ 最低存储天数说明**：冷存储和归档存储设有最低存储期限。如果对象在最低天数之前被删除，仍按最低天数收取存储费用（补收剩余天数差额）。例如：归档层对象存储 10 天后删除，仍按 180 天计费。

> **提示**：将不常访问的数据迁移到低频 / 冷 / 归档层，可大幅降低存储成本。标准存储配额内免费，非标准层按量计费无需购买套餐，单价更低。

#### 2.3.4 桶管理操作

在存储桶列表中，可对已有存储桶执行以下管理操作：

| 操作 | 说明 |
| --- | --- |
| **重命名** | 修改存储桶的自定义名称 |
| **创建快捷方式** | 在桌面上创建存储桶快捷方式，方便快速访问 |
| **删除快捷方式** | 移除桌面上的存储桶快捷方式 |
| **删除存储桶** | 删除指定的存储桶（需先清空桶内内容） |

> **⚠️ 重要提示**：删除存储桶之前必须先清空桶内的所有内容（文件和文件夹），否则删除操作将失败。删除后无法恢复。

### 2.4 存储桶详情（文件管理）

点击进入某个存储桶后，可浏览和管理桶内的所有文件与文件夹。存储桶详情页的使用方式类似个人网盘，提供完整的文件管理能力。

#### 2.4.1 文件浏览

- 以列表形式展示当前目录下的文件和文件夹
- 支持按前缀查询文件夹
- 支持搜索文件名关键字
- 当前文件夹文件过多时，仅显示前 10,000 条

#### 2.4.2 文件与文件夹管理

| 操作 | 说明 |
| --- | --- |
| **新建文件夹** | 在存储桶内创建目录结构（名称 3–30 个字符，不能包含特殊字符） |
| **上传文件** | 上传单个文件到当前目录，支持大文件自动分段上传 |
| **上传文件夹** | 上传整个文件夹（含子目录结构） |
| **下载文件** | 下载单个文件到本地（文件夹下载需通过桌面端程序） |
| **批量删除** | 选择多个文件或文件夹一次性删除 |
| **复制下载链接** | 生成文件的直接下载链接，方便分享 |

> **文件上传限制**：单次上传文件数量有上限限制，如需上传大量文件，建议打包压缩后上传，或使用 S3 专用上传工具。上传过程中可在「查看上传进度」中实时查看进度。如果文件已存在，系统会自动跳过上传。

> **文件下载提示**：网页端不支持文件夹下载，请使用桌面端程序进行下载。下载过程中请勿重复点击同一文件。

#### 2.4.3 文件编辑操作

| 操作 | 说明 |
| --- | --- |
| **重命名** | 对文件或文件夹进行重命名 |
| **复制到此** | 将文件复制到当前存储桶内的其他路径 |
| **移动到此** | 将文件移动到当前存储桶内的其他路径 |
| **转存至另一个存储桶** | 将文件复制到其他存储桶中 |
| **移动至另一个存储桶** | 将文件移动到其他存储桶中 |

> **提示**：文件较大时，复制和移动操作会自动开启加速任务，可在「存储任务」列表中查看进度。

#### 2.4.4 在线预览

存储桶支持对常见格式的文件进行在线预览，无需下载到本地即可查看文件内容。不支持预览的文件类型会提示下载后查看。

#### 2.4.5 文件分享

在存储桶详情页中，可对选中的文件进行分享：

**方式一：创建分享链接**

1. 选择需要分享的文件，点击「分享文件」或「创建链接」。
2. 设置分享参数：
   - **有效期**：限定时间（如 7 天、30 天）或永久有效
   - **提取码**：可设置密码保护，留空则无需提取码
   - **访问限制**：可设置最大访问次数（0 表示不限制）
3. 确认创建，系统生成 **预览链接** 和 **直接下载链接**。
4. 复制链接发送给其他用户。
5. 接收用户通过浏览器打开链接后，可在线预览或直接下载文件，也可将文件 **转存到自己的存储桶** 中。

**方式二：分享到论坛（博客）**

1. 选择需要分享的文件，点击「分享文件」。
2. 选择目标论坛板块（需选择至少一个技术分区）。
3. 确认分享，文件将以博客帖子的形式出现在指定板块中。

> **合规提示**：请勿侵犯他人合法权益，严禁传播色情违法内容，否则封号处理。

### 2.5 我的分享

「我的分享」页面集中管理用户创建的所有分享链接，方便统一查看、编辑和维护。

#### 2.5.1 分享链接列表

进入「我的分享」后，系统展示所有已创建的分享链接，链接状态包括：

| 状态 | 说明 |
| --- | --- |
| **有效** | 链接在有效期内，可正常访问 |
| **即将到期** | 链接即将过期 |
| **已过期** | 链接已超过有效期，无法访问 |
| **已停用** | 链接已被手动停用 |

列表页还会显示有效链接数和已过期链接数的统计信息，并支持按条件筛选。

#### 2.5.2 编辑分享链接

对已创建的分享链接可进行以下编辑操作：

| 操作 | 说明 |
| --- | --- |
| **有效期调整** | 延长或缩短分享链接的有效期，或设为永不过期 |
| **修改密码** | 输入新密码替换旧密码，或移除密码（设为公开） |
| **最大访问次数** | 设置或修改链接的访问次数限制（0 表示不限制） |
| **复制预览链接** | 复制可在浏览器打开和转存的预览链接 |
| **复制直接下载链接** | 复制文件的直接下载链接 |
| **复制密码** | 复制当前分享链接的提取码 |

#### 2.5.3 停用与重新激活

| 操作 | 说明 |
| --- | --- |
| **停用** | 暂时禁用某个分享链接，外部用户将无法访问 |
| **重新激活** | 将已停用或已过期的链接重新激活，有效期从当前时间起重新计算（默认 7 天） |

#### 2.5.4 删除分享链接

不再需要的分享链接可直接删除。

> **⚠️ 注意**：删除后不可恢复，请确认链接不再需要后再执行删除。

### 2.6 购买存储

当标准存储免费配额不足时，用户可通过「购买存储」功能扩展存储空间。

#### 2.6.1 进入方式

在存储桶列表页面点击「购买存储」或「订单购买」，进入存储空间购买页面。

#### 2.6.2 购买流程

1. **查看当前配额**：页面显示当前总存储配额的已用和可用情况。
2. **选择套餐**：从可用套餐列表中选择合适的存储空间套餐。
3. **使用优惠券**（可选）：
   - 从已有可用优惠券中选择使用
   - 或手动输入优惠券码
4. **确认订单**：查看订单汇总，确认套餐内容和金额。
5. **选择支付方式**：
   - **余额支付**：使用账户余额直接扣款
   - **微信支付**：扫码完成微信支付
6. **支付完成**：支付成功后，存储配额将在几分钟内生效。

#### 2.6.3 存储层级与计费

| 层级 | 计费方式 |
| --- | --- |
| **标准存储** | 配额内免费；超出配额后按量计费。购买套餐可增加标准层配额 |
| **低频存储** | 始终按量计费，无需购买套餐，单价低于标准存储 |
| **冷存储** | 始终按量计费，无需购买套餐，单价更低 |
| **归档存储** | 始终按量计费，无需购买套餐，单价最低 |

> **费用说明**：月费按 30 天/月估算，仅供参考，实际账单以实时存储量为准。非标准层无需单独购买套餐，创建桶时选择对应层级即可使用。

#### 2.6.4 我的订阅

在购买存储页面可查看已购买的存储订阅信息，包括有效期和调整明细。

### 2.7 存储任务

文件的复制、移动等操作在文件较大时会以异步任务方式执行。进入「存储任务」可查看所有任务的执行状态。

| 信息 | 说明 |
| --- | --- |
| **任务类型** | 文件复制任务 / 文件移动任务 |
| **任务状态** | 等待执行 / 已进入队列 / 执行中 / 已完成 |
| **创建时间** | 任务创建的时间 |
| **预计完成时间** | 系统预估的任务完成时间 |

**支持的操作**：

| 操作 | 说明 |
| --- | --- |
| **取消任务** | 取消正在执行或等待中的任务 |
| **删除记录** | 删除已完成的任务记录 |

### 2.8 密钥管理

存储管理提供 S3 兼容的访问密钥管理功能，用于第三方 S3 工具（如 S3 Browser、rclone 等）连接平台存储服务。

#### 2.8.1 存储服务信息

在存储桶列表页面可查看 S3 服务的连接信息：

| 信息 | 说明 |
| --- | --- |
| **服务地址** | S3 兼容的 API 端点地址 |
| **用户名称** | 当前用户的存储服务用户名 |

#### 2.8.2 密钥操作

| 操作 | 说明 |
| --- | --- |
| **申请新密钥** | 生成新的 S3 访问密钥（Access Key / Secret Key） |
| **查看密钥** | 查看已有密钥信息 |
| **复制密钥** | 将密钥信息复制到剪贴板 |
| **删除密钥** | 删除不再使用的密钥 |

> **⚠️ 安全提示**：请注意保护您的账号安全。删除密钥后，使用该密钥的应用将无法访问存储服务，部分应用可能无法启动，需先解除绑定。

### 2.9 挂载存储桶到应用

存储桶可挂载到运行中的应用环境，实现应用与存储桶之间的数据互通。

**典型应用场景：**

| 场景 | 说明 |
| --- | --- |
| **数据持久化** | 实验数据保存在存储桶中，即使应用被删除，数据仍保留 |
| **数据共享** | 多个应用实例共享同一份数据，无需重复上传 |
| **跨应用访问** | 在应用环境中直接读写存储桶内文件，无需额外传输 |

> 挂载操作可在应用界面的「挂载存储桶」菜单中完成，具体操作请参阅 [应用服务](./docs/application-services.md) 中的应用界面说明。

## 三、最佳实践

1. **合理规划命名空间**：为不同课程或项目创建独立的命名空间，确保资源隔离与管理清晰。
2. **合理选择存储层级**：日常频繁使用的数据放在标准存储；不常访问的备份数据迁移到低频、冷或归档层，降低存储成本。
3. **利用存储桶持久化数据**：将重要实验数据保存到存储桶，避免因应用到期或删除造成数据丢失。
4. **善用文件分享功能**：通过分享链接实现团队间的文件协作；设置密码和访问次数增强安全性；通过「我的分享」统一管理所有链接。
5. **善用桌面快捷方式**：为常用的存储桶创建桌面快捷方式，提升访问效率。
6. **关注存储配额**：定期检查存储空间使用情况，在标准配额不足时及时购买扩展。
7. **保护 S3 密钥安全**：不要将密钥分享给不信任的第三方，不再使用的密钥及时删除。
8. **删除前备份**：在删除命名空间或存储桶前，确保已对重要数据进行备份。删除存储桶前需先清空桶内所有内容。
