# 空间与存储管理

## 概述

Hi168 平台提供 **命名空间管理** 和 **存储桶管理** 两大核心资源管理能力。命名空间用于隔离不同实验或项目的运行环境与资源，避免相互冲突；存储桶则提供类似云网盘的对象存储服务，支持数据的持久化存储、分享与跨应用挂载。

---

## 一、空间管理

### 1.1 功能说明

命名空间（Namespace）是平台中资源隔离的基本单元。每个命名空间可包含特定实验或项目所需的应用、数据及其他资源，确保不同实验之间相互独立、互不干扰。

平台空间分为两类：

| 空间类型 | 说明 |
| --- | --- |
| **个人空间（私有空间）** | 用户专属的私有命名空间，拥有独立控制权，可查看已创建的应用数量及可用硬件资源 |
| **公共命名空间** | 可供多人访问和使用的共享空间，适用于团队协作场景 |

### 1.2 个人空间

进入「空间管理」→「我的个人空间」，可查看当前用户在该空间内正在运行的所有应用实例。

**支持的操作：**

- 查看应用运行状态与资源详情
- 修改应用名称
- 调整应用资源配额
- 直接打开并进入运行中的应用

### 1.3 创建个人空间

用户可在空间管理页面中创建新的个人命名空间，用于隔离不同项目或实验的资源环境。

**创建流程：**

1. 进入「空间管理」页面。
2. 点击「创建空间」。
3. 填写空间名称及相关配置。
4. 确认创建。

## 二、存储管理

### 2.1 功能说明

存储桶（Bucket）是平台提供的对象存储容器，用于存储和组织用户的数据文件。存储管理涵盖存储桶的创建、配置、内容操作、维护和监控等完整生命周期管理，确保数据安全、高效、有序地存储和访问。

### 2.2 存储桶列表

进入「存储管理」后，系统展示当前用户的所有存储桶列表，包括以下信息：

- 存储桶名称
- 类型
- 最近操作时间

### 2.3 创建存储桶

平台提供多种创建存储桶的方式：

| 方式 | 操作路径 |
| --- | --- |
| **菜单创建** | 在存储桶列表页面点击「创建存储桶」按钮 |
| **右击创建** | 在存储管理界面中右击空白区域选择创建 |
| **桌面创建** | 在桌面上直接创建存储桶 |

创建完成后，还可为存储桶在桌面上创建快捷方式，方便快速访问。

### 2.4 存储桶内容操作

存储桶的使用方式类似个人网盘，支持以下文件操作：

#### 2.4.1 文件与文件夹管理

| 操作 | 说明 |
| --- | --- |
| **新建文件夹** | 在存储桶内创建目录结构，组织文件层级 |
| **删除** | 删除存储桶中的文件或文件夹 |
| **上传** | 支持上传单个文件或整个文件夹 |
| **下载** | 下载存储桶中的文件或文件夹到本地 |
| **复制下载链接** | 生成文件的下载链接，方便分享 |

#### 2.4.2 文件编辑操作

| 操作 | 说明 |
| --- | --- |
| **复制** | 将文件复制到存储桶内的其他路径 |
| **移动** | 将文件移动到存储桶内的其他路径 |
| **重命名** | 对文件或文件夹进行重命名 |

#### 2.4.3 在线预览

存储桶支持对常见格式的文件进行在线预览，无需下载到本地即可查看文件内容。

### 2.5 分享文件

存储桶中的文件支持两种分享方式：

**方式一：分享到论坛板块**

1. 选择需要分享的文件。
2. 选择目标论坛板块。
3. 确认分享，文件将出现在指定板块中。

**方式二：分享文件链接**

1. 为目标文件生成分享链接。
2. 将链接发送给其他用户。
3. 接收用户通过浏览器打开链接后，可将文件转存到自己的存储桶中。

### 2.6 删除存储桶

在存储桶列表中可删除指定的存储桶。

> **⚠️ 重要提示**：删除存储桶之前必须先清空桶内的所有内容（文件和文件夹），否则删除操作将失败。

### 2.7 挂载存储桶到应用

存储桶可挂载到运行中的应用环境，实现应用与存储桶之间的数据互通。

**典型应用场景：**

- 实验数据的持久化保存（应用删除后数据仍保留在存储桶中）。
- 多个应用实例共享同一份数据。
- 在应用环境中直接读写存储桶内的文件。

> 挂载操作可在应用界面的「挂载存储桶」菜单中完成，具体操作请参阅 [应用服务](./application-services.md) 中的应用界面说明。

## 三、最佳实践

1. **合理规划命名空间**：为不同课程或项目创建独立的命名空间，确保资源隔离与管理清晰。
2. **利用存储桶持久化数据**：将重要实验数据保存到存储桶，避免因应用到期或删除造成数据丢失。
3. **善用文件分享功能**：通过存储桶的分享链接实现团队间的文件协作与资源共享。
4. **删除前备份**：在删除命名空间或存储桶前，确保已对重要数据进行备份。
