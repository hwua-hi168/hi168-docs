# 空间管理

## 概述

Hi168 平台提供 **命名空间管理** 能力，用于隔离不同实验或项目的运行环境与资源，避免相互冲突。每个命名空间可包含特定实验或项目所需的应用、数据及其他资源，确保不同实验之间相互独立、互不干扰。

---

## 一、功能说明

命名空间（Namespace）是平台中资源隔离的基本单元。

平台空间分为两类：

| 空间类型 | 说明 |
| --- | --- |
| **个人空间（私有空间）** | 用户专属的私有命名空间，拥有独立控制权，可查看已创建的应用数量及可用硬件资源（CPU / 内存） |
| **公共命名空间** | 可供多人访问和使用的共享空间，适用于团队协作场景 |

---

## 二、个人空间

进入「空间管理」→「我的个人空间」，可查看当前用户在该空间内正在运行的所有应用实例。

**支持的操作：**

| 操作 | 说明 |
| --- | --- |
| **查看应用状态** | 查看应用运行状态与资源详情 |
| **修改应用名称** | 直接编辑应用的显示名称 |
| **调整资源配额** | 修改应用的 CPU / 内存分配 |
| **打开应用** | 直接进入运行中的应用操作界面 |

---

## 三、创建个人空间

用户可在空间管理页面中创建新的个人命名空间，用于隔离不同项目或实验的资源环境。

**创建流程：**

1. 进入「空间管理」页面。
2. 点击「创建空间」。
3. 填写空间名称及相关配置。
4. 确认创建。

---

## 四、最佳实践

1. **合理规划命名空间**：为不同课程或项目创建独立的命名空间，确保资源隔离与管理清晰。
2. **关注资源使用**：定期查看个人空间中的应用数量和硬件资源占用情况，及时释放不需要的应用。
3. **善用公共空间**：团队协作场景下使用公共命名空间，方便多人共同访问和使用资源。
