# Notebook

## 概述

Hi168 平台的 **Notebook** 模块提供基于 Web 的交互式计算环境，支持用户在浏览器中编写和执行代码、进行数据分析与可视化、训练机器学习模型。Notebook 将代码、文档、运行结果整合在同一界面中，是数据科学与人工智能实践的核心工具。

---

## 一、功能说明

Notebook 模块允许用户创建和管理交互式计算笔记本，每个 Notebook 实例运行在独立的云端计算环境中，无需在本地安装任何开发工具或配置运行环境。

### 1.1 核心特性

| 特性 | 说明 |
| --- | --- |
| **交互式编程** | 支持逐单元格（Cell）编写和执行代码，实时查看运行结果 |
| **多语言支持** | 支持 Python 等主流编程语言的 Kernel |
| **富文本混编** | 代码单元格与 Markdown 文档单元格混合编排，实现代码与说明文档一体化 |
| **数据可视化** | 内置图表渲染能力，支持 Matplotlib、Seaborn、Plotly 等主流可视化库 |
| **云端执行** | 代码在云端计算环境中运行，本地设备无需高性能硬件 |
| **自动保存** | 编辑内容自动保存，防止意外丢失 |

## 二、Notebook 管理

### 2.1 Notebook 列表

进入 Notebook 模块后，系统展示当前用户创建的所有 Notebook 实例列表。列表页提供搜索功能，用户可通过输入名称关键字快速定位目标 Notebook。

### 2.2 创建 Notebook

点击「创建 Notebook」按钮即可创建新的 Notebook 实例：

1. 点击页面右上角「创建 Notebook」。
2. 填写 Notebook 名称及相关配置（如计算资源、Kernel 类型等）。
3. 确认创建，系统自动初始化计算环境。
4. 创建完成后，点击 Notebook 名称即可进入编辑界面。

### 2.3 编辑与运行

进入 Notebook 编辑界面后，可执行以下操作：

| 操作 | 说明 |
| --- | --- |
| **添加单元格** | 新增代码单元格或 Markdown 文档单元格 |
| **执行单元格** | 运行当前单元格中的代码，输出结果显示在单元格下方 |
| **全部执行** | 按顺序依次执行所有单元格 |
| **停止执行** | 中断正在运行的计算任务 |
| **调整顺序** | 通过拖拽或快捷键调整单元格的排列顺序 |
| **删除单元格** | 移除不需要的单元格 |

### 2.4 管理操作

| 操作 | 说明 |
| --- | --- |
| **重命名** | 修改 Notebook 的名称 |
| **删除** | 删除不再需要的 Notebook 实例及其关联的计算资源 |
| **导出** | 将 Notebook 导出为标准格式文件，便于本地保存或分享 |

## 三、典型使用场景

### 3.1 数据科学学习

- 创建包含说明文档和示例代码的 Notebook，分享给团队成员。
- 在 Notebook 中逐步执行代码，理解数据处理与分析流程。

### 3.2 模型训练与实验

- 在 Notebook 中加载平台数据集，进行数据预处理。
- 编写模型训练代码并实时观察训练过程和指标变化。
- 利用可视化工具展示训练结果和模型性能。

### 3.3 数据探索与分析

- 加载业务数据集进行探索性数据分析（EDA）。
- 使用图表和统计方法发现数据规律与潜在问题。
- 将分析过程和结论整理为可复现的 Notebook 文档。

---

## 四、最佳实践

1. **合理组织单元格**：将代码按逻辑分段，配合 Markdown 单元格添加说明文字，提升可读性。
2. **配合数据集模块使用**：直接引用平台数据集，避免在 Notebook 中硬编码数据路径。
3. **及时保存与导出**：虽然平台支持自动保存，但建议定期导出重要 Notebook 作为备份。
4. **注意资源释放**：长时间不使用的 Notebook 建议关闭或删除，以释放云端计算资源。
