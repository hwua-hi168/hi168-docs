# 模板与镜像管理

## 概述

Hi168 平台提供完善的 **模板管理** 和 **镜像管理** 功能，构成应用环境复用与分发的核心基础设施。用户可将已配置好的应用环境保存为模板进行复用与共享，也可管理底层的容器镜像和虚拟机镜像，实现从镜像到模板再到应用的完整工作流。

---

## 一、我的模板

### 1.1 功能说明

「我的模板」模块用于保存和管理用户自定义的应用模板。用户可将常用的应用环境配置保存为模板，并在需要时快速部署新的应用实例。模板还支持编辑服务端口、算力配置等参数，并可发布到应用中心或分享给指定用户。

### 1.2 模板列表

进入「我的模板」后，系统列出当前用户创建的所有应用模板，模板状态分为：

| 状态 | 说明 |
| --- | --- |
| **制作中** | 模板正在创建过程中，尚未完成 |
| **已发布** | 模板已发布到应用中心，供其他用户使用 |

在模板列表中可执行以下操作：查询、删除、发布、编辑、部署、分享。

### 1.3 编辑模板

在发布模板之前或之后，均可对模板内容进行编辑修改。编辑范围包括：

- 模板基本信息（名称、描述等）
- 服务端口配置
- 算力资源配置
- 功能支持选项

### 1.4 发布模板

模板制作完成后，用户可将其发布到应用中心，使所有平台用户均可搜索和使用该模板。

**发布流程：**

1. 在模板列表中选择目标模板。
2. 点击「发布」按钮。
3. 填写发布信息并确认提交。
4. 审核通过后，模板将在应用中心上架展示。

> **建议**：在正式发布前，先通过「部署」功能进行测试，确保模板部署后运行正常。

### 1.5 分享模板

如果不希望将模板公开发布到应用中心，也可选择将模板分享给指定用户或所有用户。分享后，模板将出现在目标用户的「我的模板」列表中。

**分享选项：**

- **分享对象**：指定用户或全部用户。
- **有效期**：可选择「限定时间」或「长期有效」。

### 1.6 部署模板

在「我的模板」中部署模板的流程与在应用中心部署应用一致，区别仅在于入口不同。典型应用场景包括：

- 对新制作的模板进行部署测试，验证环境完整性。
- 基于现有模板快速创建新的应用实例。

## 二、我的镜像

### 2.1 功能说明

「我的镜像」模块提供容器镜像与虚拟机镜像的统一管理能力，支持镜像的创建、查看、编辑、删除操作，并可基于镜像制作应用模板。

### 2.2 容器镜像管理

#### 2.2.1 容器镜像列表

进入「我的镜像」后，默认展示容器镜像列表，每条记录包含以下信息：

- 镜像 ID
- 镜像名称
- 仓库地址
- 标签（Tag）
- 描述
- 创建时间
- 创建人

#### 2.2.2 容器镜像详情

点击镜像名称可进入详情页，查看镜像的完整元数据信息。在详情页中同时支持：

- **编辑镜像**：修改镜像名称、描述等基本信息。
- **删除镜像**：从平台中移除该镜像。

#### 2.2.3 新建容器镜像

在容器镜像列表页点击「新建容器镜像」，可从指定的容器镜像仓库拉取镜像到本地。拉取成功后，镜像将同步到平台本地仓库，后续可基于该镜像制作应用模板。

**操作步骤：**

1. 点击「新建容器镜像」进入创建页面。
2. 填写镜像来源信息（仓库地址、镜像名称、标签等）。
3. 确认提交，系统将自动从远程仓库拉取镜像。
4. 拉取成功后，镜像出现在列表中，可用于后续的模板制作。

### 2.3 虚拟机镜像管理

#### 2.3.1 虚拟机镜像列表

在「我的镜像」中切换到「虚拟机镜像」标签，展示所有虚拟机镜像记录，每条记录包含：

- 镜像 ID
- 镜像名称
- 代号
- 描述
- 状态
- 创建时间
- 创建人

#### 2.3.2 虚拟机镜像详情

点击镜像名称进入详情页，查看虚拟机镜像的完整信息。支持在详情页中编辑镜像的基本信息。

#### 2.3.3 新建虚拟机镜像与制作应用模板

平台支持创建虚拟机镜像并基于镜像制作应用模板的完整流程：

**第一步：新建虚拟机镜像**

1. 在虚拟机镜像列表页点击「新建虚拟机镜像」。
2. 填写镜像基本信息（名称、代号、描述等）。
3. 选择或上传虚拟机镜像文件。
4. 确认提交，等待镜像创建完成。

**第二步：制作应用模板**

1. 镜像创建完成后，点击「制作应用模板」。
2. 配置模板参数（模板名称、介绍、服务端口、资源配额等）。
3. 确认提交，系统将基于镜像生成应用模板。

**第三步：部署应用模板**

模板制作完成后，可通过应用模板详情页或「我的模板」直接进行部署，流程与应用中心的标准部署流程一致。

---

## 三、工作流示意

模板与镜像管理模块支持以下典型工作流：

```
镜像拉取/上传 → 镜像管理 → 制作应用模板 → 编辑模板 → 部署测试 → 发布/分享
```

1. **镜像准备**：从外部仓库拉取容器镜像，或上传虚拟机镜像到平台。
2. **模板制作**：基于镜像或已有应用的快照创建应用模板。
3. **模板调优**：编辑模板的配置参数，确保满足实际需求。
4. **部署验证**：在正式发布前，通过部署测试验证模板的完整性和正确性。
5. **发布与分享**：将验证通过的模板发布到应用中心或分享给指定用户。

---

## 四、注意事项

- 模板制作时间取决于镜像内容的大小，大型镜像可能需要较长的处理时间。
- 建议在发布前始终进行部署测试，确保终端用户获得完整可用的应用环境。
- 删除镜像前请确认不存在依赖该镜像的模板或运行中的应用。
- 分享模板时需注意有效期设置，避免过期后影响接收用户的使用。
