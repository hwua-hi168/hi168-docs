# 附录

## 计费规则

### 一、余额扣减机制

Hi168 平台采用 **余额** 作为资源计费单位。用户在部署和运行应用时，系统将根据所选资源规格按规则扣减余额。

### 二、扣费规则

#### 2.1 部署阶段扣费

| 阶段 | 扣费方式 |
| --- | --- |
| **首次部署** | 应用部署时根据所选规格（CPU / 内存）进行首次余额扣减 |
| **运行期间** | 部署成功后，系统每小时按部署时的余额标准进行持续扣减 |

#### 2.2 停机阶段扣费（暂行）

| 阶段 | 扣费方式 |
| --- | --- |
| **应用停机后** | 按应用中的存储占用大小进行余额扣减 |

### 三、计费说明

- 不同资源规格对应不同的余额扣减标准，规格越高、扣减越多。
- 应用运行期间的扣费为周期性扣减（每小时一次）。
- 应用停机后仍会因存储资源占用产生少量费用。
- 彻底删除应用并释放存储后，将停止一切扣费。

### 四、费用管理建议

1. **按需部署**：根据实际需要选择合适的资源规格，避免资源过度分配导致余额浪费。
2. **及时清理**：不再使用的应用建议及时删除，释放存储资源并停止扣费。
3. **关注余额**：定期在「费用中心」查看余额和消耗明细，合理规划使用。
4. **善用快照**：对于临时性实验，可创建快照保存状态后删除应用，需要时通过快照恢复，减少持续运行的算力消耗。
