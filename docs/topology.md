# 拓扑管理

## 概述

Hi168 平台的 **拓扑管理** 模块提供可视化的网络与基础设施拓扑编排能力。用户可通过图形化界面设计、构建和管理复杂的网络拓扑结构，直观地编排服务器、交换机、路由器、防火墙等各类基础设施组件及其互联关系，广泛应用于网络工程实践、实验环境搭建和架构设计等场景。

---

## 一、功能说明

拓扑管理采用所见即所得的可视化编辑模式，将抽象的网络架构转化为直观的图形表示，降低拓扑设计的复杂度。

### 1.1 核心特性

| 特性 | 说明 |
| --- | --- |
| **可视化画布** | 提供无限缩放的网格画布，支持拖拽式拓扑设计 |
| **丰富组件库** | 内置服务器、交换机、路由器、防火墙等多种网络设备组件 |
| **连线与互联** | 通过连线工具定义组件之间的网络连接关系 |
| **缩放与导航** | 支持画布的缩放、平移和全局视图导航 |
| **多种视图模式** | 支持网格视图、列表视图等不同展示方式 |
| **实时部署** | 编排完成后可一键部署拓扑中的所有节点和网络配置 |

---

## 二、界面布局

### 2.1 工具栏

拓扑编辑界面顶部提供完整的工具栏，包含以下功能区域：

**操作工具：**

| 工具 | 说明 |
| --- | --- |
| **启动** | 启动当前拓扑中的所有节点 |
| **停止** | 停止当前拓扑中所有运行中的节点 |
| **连接** | 在两个组件之间建立网络连接 |
| **网络配置** | 配置拓扑中的网络参数（子网、VLAN 等） |

**绘图工具：**

| 工具 | 说明 |
| --- | --- |
| **矩形** | 在画布上绘制矩形区域，用于标注逻辑分组 |
| **线条** | 绘制辅助线条，标注网络路径或逻辑关系 |
| **网格模式** | 切换网格显示，辅助组件对齐 |
| **分屏模式** | 分屏查看拓扑的不同区域 |

**视图控制：**

| 工具 | 说明 |
| --- | --- |
| **放大 / 缩小** | 调整画布的显示缩放比例 |
| **适配视图** | 自动调整缩放，使所有组件完整显示 |
| **全屏显示** | 进入全屏编辑模式 |
| **设置** | 配置拓扑的全局参数 |
| **刷新** | 刷新拓扑状态 |

### 2.2 组件面板

界面左侧为组件面板，提供可拖拽的基础设施组件：

| 组件类型 | 说明 |
| --- | --- |
| **服务器** | 物理服务器或虚拟机节点 |
| **终端设备** | PC、笔记本等终端设备 |
| **交换机** | 二层交换设备，用于局域网组网 |
| **路由器 / 防火墙** | 三层路由设备或安全设备 |
| **更多组件** | 通过扩展菜单获取更多设备类型 |

### 2.3 拓扑画布

中央区域为拓扑画布，用户在此进行拓扑的可视化编排：

- **拖放组件**：从左侧面板拖拽组件到画布中放置。
- **连接组件**：选择连接工具，在两个组件之间拖动建立连线。
- **移动与排列**：拖动已放置的组件调整位置，优化拓扑布局。
- **属性编辑**：双击组件可编辑其属性参数（IP 地址、主机名等）。

## 三、拓扑管理操作

### 3.1 创建拓扑

1. 进入拓扑管理模块。
2. 点击「新建拓扑」创建一个新的拓扑项目。
3. 为拓扑命名并设置基本参数。

### 3.2 编排拓扑

1. 从组件面板中拖拽所需设备到画布。
2. 使用连接工具建立设备之间的网络互联。
3. 双击设备配置网络参数（IP、子网掩码、网关等）。
4. 使用绘图工具添加逻辑分组和标注。

### 3.3 部署与运行

1. 拓扑编排完成后，点击「启动」按钮。
2. 系统按照拓扑定义自动创建和配置所有节点。
3. 节点启动后，用户可点击节点进入其操作界面。
4. 需要停止时，点击「停止」按钮关闭所有节点。

### 3.4 管理拓扑

| 操作 | 说明 |
| --- | --- |
| **编辑** | 修改已有拓扑的结构和配置 |
| **复制** | 基于现有拓扑复制一个新的副本 |
| **删除** | 删除不再需要的拓扑项目 |
| **导出** | 将拓扑结构导出为文件，便于备份或迁移 |

## 四、典型使用场景

### 4.1 网络工程实践

- 设计网络拓扑实验，通过可视化界面理解网络架构。
- 搭建包含路由器、交换机、防火墙的企业级网络实验环境。
- 实践 VLAN 划分、路由协议配置、ACL 策略设置等网络实验。

### 4.2 系统架构设计

- 构建微服务架构的拓扑示意图，规划服务间的网络通信。
- 设计高可用集群的部署拓扑，验证故障转移策略。

### 4.3 网络安全实验

- 搭建包含攻防环境的网络拓扑，开展渗透测试实训。
- 配置防火墙规则，实践网络安全策略的部署与验证。

---

## 五、最佳实践

1. **先规划后编排**：在开始拖拽之前，先明确拓扑的整体架构和网络分段策略。
2. **合理使用分组**：利用矩形标注工具将逻辑相关的设备分组，提升拓扑可读性。
3. **规范命名**：为每个节点设置有意义的名称，方便在复杂拓扑中快速识别。
4. **保存与备份**：定期保存拓扑编排结果，重要拓扑建议导出备份。
