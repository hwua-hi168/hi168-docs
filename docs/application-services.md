# 应用服务

## 概述

Hi168 平台的 **应用服务** 模块是用户使用平台的核心入口，涵盖应用的浏览、部署、运行及全生命周期管理。平台提供丰富的预置应用模板，覆盖网络安全、人工智能、大数据、开发运维、操作系统等多个领域，用户可一键部署所需的实验或工作环境，并通过「我的应用」对已部署的实例进行统一管理。

---

## 一、应用中心

### 1.1 功能说明

应用中心是平台所有可用应用模板的集中展示区域。每个应用模板定义了部署一个完整实验环境的全部流程，用户选择并使用某个应用后，即可一键搭建对应的实验或工作环境。

### 1.2 应用列表

进入「应用中心」后，系统将以列表形式展示所有可用的应用模板，并提供多维度筛选条件，方便用户快速定位目标应用：

**按应用类型筛选：**

- **容器**：基于容器技术的轻量级应用，启动速度快、资源占用低。
- **虚拟机**：基于虚拟化技术的完整操作系统实例，适用于需要图形化桌面或完整系统环境的场景。

**按应用类别筛选：**

- 网络安全、游戏、娱乐影音、办公生活、人工智能、开发、网络运维、操作系统、大数据

**按操作系统筛选：**

- Linux、Windows

**按架构筛选：**

- AMD

此外，用户也可通过搜索框输入关键字进行模糊检索。应用列表实时更新，展示最新和最热门的应用供用户选择。

### 1.3 应用部署

应用部署的标准流程如下：

1. **选择应用**：在应用中心列表中浏览或搜索目标应用，点击进入应用详情。
2. **配置部署参数**：根据需求选择命名空间、资源配额（CPU / 内存）等部署选项。
3. **确认部署**：点击「部署」按钮，系统将自动完成环境的创建与初始化。
4. **运行应用**：部署完成后，应用进入运行状态，用户可直接进入应用环境进行操作。

> **说明**：部署应用会消耗余额，具体扣减规则请参阅 [计费规则](./appendix.md)。

### 1.4 应用界面

Hi168 的应用分为 **容器应用** 和 **虚拟机应用** 两种类型，两者的操作界面菜单基本一致，但在部分功能上存在差异。

#### 1.4.1 容器应用界面

容器应用（如 Nginx）的操作界面提供以下功能菜单：

| 功能 | 说明 |
| --- | --- |
| **HTTP Web 服务** | 以浏览器方式打开指定端口的 HTTP Web 服务，需提前在「服务端口」中配置对外暴露的端口 |
| **命令行终端** | 提供 Linux 命令行终端，支持常规的 Shell 操作 |
| **服务端口** | 管理网络端口暴露配置，支持 HTTP、TCP、UDP、TTYD、WEBSHELL 五种服务类型。例如：可将 MySQL 的 3306 端口暴露给外部工具连接使用 |
| **挂载存储桶** | 将已有存储桶挂载到应用环境中，实现数据持久化存储与跨应用共享 |
| **文件系统** | 提供图形化的 Linux 文件管理器，支持文件上传、下载等操作 |
| **刷新** | 刷新当前应用界面 |
| **应用快照** | 为当前应用状态创建快照，支持后续的状态恢复与模板制作 |

#### 1.4.2 虚拟机应用界面

虚拟机应用（如 Linux 云桌面）的操作界面在容器应用基础上增加了以下功能：

| 功能 | 说明 |
| --- | --- |
| **图形化终端** | 通过 VNC 协议连接应用环境中的图形化桌面界面 |
| **命令行终端** | 同容器应用 |
| **服务端口** | 同容器应用 |
| **挂载存储桶** | 同容器应用；虚拟机应用支持将存储桶挂载到指定的系统路径 |
| **文件上传下载** | 通过 `trz`（上传）和 `tsz`（下载）命令进行文件传输 |
| **刷新** | 同容器应用 |
| **控制面板** | noVNC 自带的控制面板，提供剪贴板等辅助功能（可将外部文本复制到虚拟机中） |
| **应用快照** | 同容器应用 |
| **自动调整界面比例** | 自适应调整图形化界面的长宽比例，优化显示效果 |

## 二、我的应用

### 2.1 功能说明

「我的应用」模块用于管理当前用户已部署的所有应用实例。用户可在此查看应用的运行状态、资源消耗、到期时间等关键信息，并执行日常管理操作。

### 2.2 应用列表

进入「我的应用」后，系统按部署时间倒序展示当前用户所有正在运行的应用，列表中包含以下信息：

- 用户总余额与当前应用消耗总额
- 每个应用的创建时间、到期时间
- 应用资源配额（CPU / 内存）

用户也可直接在此页面通过「添加应用」入口快速跳转至应用中心进行新应用的部署。

### 2.3 应用详情

点击某个应用，可进入其详情页查看完整信息：

- **应用概览**：应用 ID、名称、模板名称、运行状态、创建时间、到期时间、命名空间、资源配额、余额消耗、应用描述。
- **环境列表**：与该应用关联的环境实例。
- **快照列表**：已为该应用创建的所有快照记录。

> 应用名称、环境名称和资源配额支持在详情页直接修改。

### 2.4 修改配额

在应用使用过程中，如发现当前配置不足以满足性能需求，可通过「修改配额」功能提升应用的 CPU 或内存资源分配，优化运行性能。

### 2.5 创建快照

快照（Snapshot）功能允许用户在任意时刻保存应用的当前状态与数据，以便后续恢复或共享。

**支持的操作：**

- **创建快照**：在应用管理界面中点击「创建快照」，系统将按照当前环境状态生成快照并添加到快照列表。
- **删除快照**：在快照列表中选择目标快照执行删除操作。
- **回滚快照**：将应用环境恢复至某个快照所记录的状态。
- **基于快照创建模板**：以当前快照为基础创建应用模板，供后续复用。

### 2.6 创建模板

用户可将当前应用的状态和配置保存为模板（Template），方便后续快速创建相似环境。

**创建步骤：**

1. 在应用概览界面点击「创建模板」。
2. 填写模板名称（必填）。
3. 上传模板展示图片（可选），便于直观识别模板用途。
4. 填写模板介绍（必填），描述模板包含的软件环境和适用场景。
5. 配置功能支持选项：
   - **支持 ttyd**：选择「不变 / 是 / 否」。
   - **支持 webshell**：选择「不变 / 是 / 否」。
6. 确认信息无误后点击「确定」，系统开始制作模板。

> 模板制作耗时取决于镜像修改内容的大小，可能从数秒到数分钟不等。制作进度可在快照列表中查看，完成后前往「我的模板」查看结果。

### 2.7 删除应用

平台提供两种删除应用的方式：

- **方式一**：在「应用概览」详情页中执行删除操作。
- **方式二**：在桌面上右击应用图标，选择「删除」。

> **注意**：删除操作不可恢复，请确认应用中的重要数据已通过快照或存储桶进行备份。

## 三、最佳实践

1. **部署前确认资源配额**：根据应用的实际资源需求选择合适的 CPU 和内存配置，避免因资源不足导致运行异常。
2. **定期创建快照**：在进行关键实验操作前创建快照，确保可随时回退到安全状态。
3. **善用模板功能**：将配置好的环境保存为模板，可在后续实验中快速复用，显著提升效率。
4. **关注余额消耗**：应用运行期间持续消耗余额，不再使用的应用建议及时删除以节省资源。
